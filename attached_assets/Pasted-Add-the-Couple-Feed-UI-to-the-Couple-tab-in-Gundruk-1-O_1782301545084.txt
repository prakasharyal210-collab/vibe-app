Add the Couple Feed UI to the Couple tab in Gundruk.

1. On the Couple home screen (artifacts/mobile/app/(tabs)/find.tsx couple section, or wherever the couple cards are), add a new card at the top (above Couple of the Month):
   - "💬 Couple Feed" / "Share your story with other couples"
   - Tapping it navigates to /couple/feed

2. Create artifacts/mobile/app/couple/feed.tsx — the main feed screen:
   - Header: "Couple Feed 💕"
   - A "+ Share" button (top right) to create a new post
   - Category filter chips at top: All / Story / Advice / Milestone / Venting
   - Scrollable list of posts. Each post card shows:
     - Both partner avatars (side by side, overlapping slightly) + couple name "Prakash & Priya"
     - Category badge (colored pill)
     - Post text content
     - Photo if photo_url exists
     - ❤️ like button with count (filled red if likedByMe) + 💬 comment button with count
     - Time ago (e.g. "2h ago")
   - Pull to refresh
   - Calls GET /api/couple-feed/posts?coupleId=<myCoupleId>

3. Create artifacts/mobile/app/couple/feed-create.tsx — create post screen/modal:
   - Text input (multiline, placeholder "Share your story, feelings, or ask for advice...")
   - Category selector: Story / Advice / Milestone / Venting
   - Optional photo picker (use existing image upload util in the app)
   - "Post" button → POST /api/couple-feed/posts with { coupleId, authorId, content, photoUrl, category }
   - On success, go back to feed and refresh

4. Create artifacts/mobile/app/couple/feed-comments.tsx — comments screen for a post:
   - Shows the original post at top
   - List of comments with author avatar + name + text + time
   - Text input at bottom to add a comment → POST /api/couple-feed/posts/:postId/comments
   - Calls GET /api/couple-feed/posts/:postId/comments

5. Like behavior: tapping ❤️ calls POST .../like, tapping again calls DELETE .../like. Update count optimistically.

Use the CoupleContext to get the current coupleId and the auth context for authorId.
All API calls use process.env.EXPO_PUBLIC_API_URL and pass Authorization Bearer token.
Style: dark theme, pink accents (#EC4899) for couple features, category badges in distinct colors.
