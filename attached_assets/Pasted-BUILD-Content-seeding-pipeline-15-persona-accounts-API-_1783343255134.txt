DELIVERABLE 2 — scripts/seed-content.ts (in artifacts/api-server or scripts/):
A Node script that reads scripts/seed-content-batch.json and posts each item through the EXISTING api-server endpoints (running server required) — NOT direct supabase inserts — so moderation, counters, categories, and rendering all work exactly like real posts. Supports item types: post (caption + optional image URL from Pexels API — key already in env — searched by a query field per item), poll (question + 2-4 options, feed or confession), confession (couple personas only, with their couple context), comment (personaId comments on another seed post by index reference), like (persona likes a seed post). Each item: { personaUsername, type, payload, scheduleHint }. Script resolves persona userIds by username via the DB, posts sequentially with small random delays, logs each result, { data, error } checked everywhere, summary table at the end (created/failed). Add a --dry-run flag that validates the JSON and resolves personas without posting.
DELIVERABLE 3 — scripts/generate-seed-content.ts:
Uses the ANTHROPIC_API_KEY (same pattern as the moderation/welcome code) to generate seed-content-batch.json: takes the 15 persona definitions (embed their personality sheets in this script as the single source of truth), asks Claude to write 7 days of content: ~6-8 items/day mixed across personas — posts in each persona's distinct voice (Nepali-English code-switching where natural, "yo semester is killing me yaar" energy), 2-3 polls (fun debates: momo vs chowmein, arranged vs love marriage opinions — keep it warm not divisive), 2 couple confessions (relatable, wholesome-to-bittersweet range), and inter-persona comments so threads feel alive. Output valid JSON matching Deliverable 2's schema. Content rules baked into the generation prompt: no politics, no real people, nothing that would trip our own moderation, feels like real 20-somethings not marketing copy.
