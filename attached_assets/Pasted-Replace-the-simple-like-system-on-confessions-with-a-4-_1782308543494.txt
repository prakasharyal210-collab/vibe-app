Replace the simple like system on confessions with a 4-reaction system (like Facebook reactions but for emotional support).

THE 4 REACTIONS:
- support (🫂) - "I'm here for you"
- relate (🥲) - "I feel this"
- strength (💪) - "Stay strong"
- love (❤️) - "This touched me"

STEP 1 — DATABASE (give me SQL to run in Supabase first):
The couple_feed_likes table currently stores one like per couple per post. Convert it to store a reaction type:

ALTER TABLE couple_feed_likes ADD COLUMN IF NOT EXISTS reaction TEXT DEFAULT 'support' CHECK (reaction IN ('support', 'relate', 'strength', 'love'));

Also we need per-reaction counts on posts. Instead of just like_count, add:
ALTER TABLE couple_feed_posts ADD COLUMN IF NOT EXISTS reaction_support INT DEFAULT 0;
ALTER TABLE couple_feed_posts ADD COLUMN IF NOT EXISTS reaction_relate INT DEFAULT 0;
ALTER TABLE couple_feed_posts ADD COLUMN IF NOT EXISTS reaction_strength INT DEFAULT 0;
ALTER TABLE couple_feed_posts ADD COLUMN IF NOT EXISTS reaction_love INT DEFAULT 0;

(keep like_count as the total of all reactions for backwards compat)

STEP 2 — API (artifacts/api-server/src/routes/couple-feed.ts):
- POST /api/couple-feed/posts/:postId/react
  - Body: { coupleId, likerId, reaction } where reaction is one of support/relate/strength/love
  - If the couple already reacted: update their reaction to the new type (and adjust the per-reaction counts: decrement old reaction count, increment new one)
  - If new reaction: insert row, increment that reaction's count + like_count
  - Return updated counts + myReaction
- DELETE /api/couple-feed/posts/:postId/react
  - Body: { coupleId }
  - Remove the couple's reaction, decrement the appropriate count + like_count
  - Return updated counts
- GET /api/couple-feed/posts now returns for each post:
  - reactions: { support, relate, strength, love } counts
  - totalReactions (sum)
  - myReaction: 'support' | 'relate' | 'strength' | 'love' | null (what THIS couple reacted, based on coupleId param)
