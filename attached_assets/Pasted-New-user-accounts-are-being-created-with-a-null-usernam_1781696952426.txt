New user accounts are being created with a null username in the profiles table. Confirmed in the database — a newly signed-up account has username = null (and also age = null, show_in_matching = false). Example row: id 81422457-3e4b-44bf-9204-a55f9254e400 has username = null.
This breaks multiple things: the profile can't be looked up by username, it won't render properly in feeds/matching, and Find Vibe filtering may exclude it.

Is the insert a direct Supabase call that's hanging?
